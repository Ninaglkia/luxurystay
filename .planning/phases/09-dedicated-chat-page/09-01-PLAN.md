```typescript
  )
}
```

Critical implementation details:
- `h-dvh` (not `100vh`) — dvh accounts for iOS Safari virtual keyboard; Tailwind 4 supports this natively
- `overflow-hidden` on main — prevents double scrollbar; ChatMessages owns its own scroll container
- `flex-none` on header — header does not grow or shrink; takes only its natural height
- `flex-1 min-h-0` on the ChatWidget wrapper — `flex-1` makes it fill remaining height; `min-h-0` is CRITICAL: without it, the default `min-height: auto` on a flex child prevents `overflow-y-auto` from activating on grandchildren (ChatMessages), causing the whole page to scroll instead of the message list
- `ChatWidget className="h-full"` — threads className to ChatWidget's root div which uses it as `${className ?? ''}`, making ChatWidget fill the wrapper height
- No `propertyId` prop — this is the general concierge page (not property-specific)
- No border, shadow, or rounded corners — this is a full-screen page, not a card
- This is a Server Component (no 'use client') — metadata export requires it

Modify src/app/components/chat-messages.tsx to add auto-scroll using a useRef sentinel pattern. The existing rendering logic (message bubbles, streaming indicator) must be preserved exactly — only add the ref, effect, and sentinel element:

```typescript
'use client'

import { useEffect, useRef } from 'react'
import type { UIMessage } from 'ai'

interface ChatMessagesProps {
  messages: UIMessage[]
  isStreaming: boolean
}

export default function ChatMessages({ messages, isStreaming }: ChatMessagesProps) {
  const bottomRef = useRef<HTMLDivElement>(null)

  useEffect(() => {
    bottomRef.current?.scrollIntoView({ behavior: 'smooth' })
  }, [messages, isStreaming])

  return (
    <div className="flex-1 overflow-y-auto p-4 space-y-4">
      {messages.map((message) => (
        <div
          key={message.id}
          className={`flex ${message.role === 'user' ? 'justify-end' : 'justify-start'}`}
        >
          <div
            className={`max-w-[80%] rounded-2xl px-4 py-3 text-sm leading-relaxed ${
              message.role === 'user'
                ? 'bg-neutral-900 text-white'
                : 'bg-neutral-100 text-neutral-900'
            }`}
          >
            {message.parts
              .filter((p): p is { type: 'text'; text: string } => p.type === 'text')
              .map((p, i) => (
                <span key={i}>{p.text}</span>
              ))}
          </div>
        </div>
      ))}
      {isStreaming && (
        <div className="flex justify-start">
          <div className="bg-neutral-100 text-neutral-400 rounded-2xl px-4 py-3 text-sm">
            <span className="animate-pulse">...</span>
          </div>
        </div>
      )}
      {/* Sentinel — always rendered, always at the bottom of the list */}
      <div ref={bottomRef} aria-hidden="true" />
    </div>
  )
}
```

Implementation notes:
- `useEffect` dependency array includes both `messages` AND `isStreaming` — scroll fires when a new message is added (user sends) and when streaming ends (AI response completes). This is intentional.
- `behavior: 'smooth'` is the starting choice. If jitter is observed during streaming (because @ai-sdk/react v3 updates messages array reference on every token), change to `behavior: 'instant'` — a one-word fix.
- The sentinel `<div ref={bottomRef} aria-hidden="true" />` is unconditional — always in the DOM so scrollIntoView always has a target. No timing issue possible.
- `useRef<HTMLDivElement>(null)` — typed correctly for a div element.
- The sentinel approach applies auto-scroll universally — both the /chat page AND the ChatBubble overlay will auto-scroll. This is correct behavior: users in the bubble also want to see the latest message.
- Do NOT add an `autoScroll` prop at this time — requirements do not ask for conditional scroll behavior. The universal behavior is correct.
