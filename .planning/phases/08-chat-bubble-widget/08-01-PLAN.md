  <action>
Create `src/app/components/chat-bubble.tsx` as a `'use client'` component. This is the main widget: a floating action button that toggles an animated chat overlay.

**Implementation details:**

1. Import `useState` from react, `AnimatePresence` and `motion` from `'framer-motion'` (not 'motion/react'), `usePathname` from `'next/navigation'`, and `ChatWidget` from `'./chat-widget'`.

2. Guard: if `pathname === '/chat'` return null — the dedicated chat page does not need the bubble.

3. State: `const [isOpen, setIsOpen] = useState(false)`.

4. Render two sibling elements wrapped in a React Fragment (`<>`):
   a. **FAB button** — fixed position, bottom-right, always visible:
      ```
      className="fixed z-50 w-14 h-14 rounded-full
                 bg-neutral-900 text-white shadow-lg
                 flex items-center justify-center
                 hover:bg-neutral-700 transition-colors
                 focus-visible:ring-2 focus-visible:ring-offset-2 focus-visible:ring-neutral-900"
      style={{ bottom: 'max(1.5rem, env(safe-area-inset-bottom, 1.5rem))', right: '1.5rem' }}
      aria-label={isOpen ? 'Chiudi chat' : 'Apri chat'}
      onClick={() => setIsOpen((prev) => !prev)}
      ```
      Button content: when `isOpen`, render a close X icon (inline SVG); when closed, render a chat bubble icon (inline SVG). Use simple inline SVG paths — no icon library. Chat icon: speech bubble shape (24x24 viewBox). X icon: two crossing lines (standard close pattern). Both icons: `currentColor` fill/stroke, `w-6 h-6`.

   b. **AnimatePresence overlay** — conditionally mounts ChatWidget:
      ```tsx
      <AnimatePresence>
        {isOpen && (
          <motion.div
            key="chat-overlay"
            initial={{ opacity: 0, y: '100%' }}
            animate={{ opacity: 1, y: 0 }}
            exit={{ opacity: 0, y: '100%' }}
            transition={{ type: 'spring', damping: 30, stiffness: 300 }}
            className="fixed right-6 z-40
                       w-[calc(100vw-3rem)] max-w-sm
                       h-[min(600px,80dvh)]
                       rounded-2xl shadow-2xl overflow-hidden
                       border border-neutral-200"
            style={{ bottom: 'max(5.5rem, calc(env(safe-area-inset-bottom, 0px) + 5.5rem))' }}
          >
            <ChatWidget />
          </motion.div>
        )}
      </AnimatePresence>
      ```
      **Critical:** `ChatWidget` is ONLY rendered inside the `{isOpen && ...}` block. Never render it outside — that would load it on page startup and defeat lazy loading.
      **Critical:** `key="chat-overlay"` on the `motion.div` is required for exit animations to fire.
      **Critical:** Use `80dvh` (dynamic viewport height) NOT `80vh` — `dvh` shrinks when the iOS keyboard opens, keeping the input visible.
  </action>

**Part A — Create `src/app/components/chat-bubble-wrapper.tsx`:**

This is a thin `'use client'` shim whose sole purpose is to hold the `next/dynamic` call with `ssr: false`. Server Components cannot use `ssr: false` — this wrapper provides the client boundary.

```typescript
'use client'

import dynamic from 'next/dynamic'

const ChatBubble = dynamic(() => import('./chat-bubble'), {
  ssr: false,
  loading: () => null, // render nothing while JS chunk loads — no layout shift
})

export default function ChatBubbleWrapper() {
  return <ChatBubble />
}
```

No other logic goes here. This file stays minimal.

**Part B — Update `src/app/layout.tsx`:**

Add a single import and a single JSX element. `layout.tsx` stays a Server Component — do NOT add `'use client'`.

1. Add import at the top: `import ChatBubbleWrapper from './components/chat-bubble-wrapper'`
2. Inside `<body>`, after `{children}` and before the `<script dangerouslySetInnerHTML=...>` service worker tag, add: `<ChatBubbleWrapper />`

The final body should look like:

<body className={`${geistSans.variable} ${geistMono.variable} antialiased`}>

Do not move or modify the existing service worker script. Do not add 'use client' to layout.tsx. Do not change any metadata, font config, or head tags.
