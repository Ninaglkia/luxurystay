<objective>
Fix a security bug in proxy.ts where a client-supplied x-user-id header can bypass auth tier detection, then verify all four access-control requirements with tests.

Purpose: AUTH-03 is the critical invariant — the auth tier must be unforgeable. A client that sends x-user-id in the request currently has that header forwarded to route.ts when they are not authenticated (because the proxy only adds x-user-id inside `if (user)`, it never removes it). Phase 4+ will rely on this invariant being correct.

Output: Fixed proxy.ts (header stripping on all requests) + four new tests in proxy.test.ts covering AUTH-01 through AUTH-04.
